## 배열과 리스트

### 배열
+ 같은 종류의 데이터를 `연속된 메모리 공간`에 저장하는 자료구조
+ 각 데이터 요소는 인덱스를 통해 직접 접근이 가능하며 `고정된 크기`를 가짐
+ 인덱스를 이용하여 원하는 요소에 O(1) 시간 복잡도로 접근 가능
+ 연속된 메모리 배치로 캐시 효율성이 높아 속도가 빠름
+ 크기가 고정되어 있어 배열의 길이를 넘도록 추가할 수 없지만 범위 내에서 삽입/삭제는 가능하다
    - O(n)의 시간복잡도

    ```
    int[] arr = new int[5];

    arr[0] = 10;
    arr[1] = 20;
    arr[2] = 30;
    arr[3] = 40;
    arr[4] = 50;
    ```

### 리스트
+ 순차적으로 데이터를 저장하는 자료구조
+ 배열과 달리 `동적으로 크기가 조정`가능하며 삽입/삭제가 가능
    - 배열과 달리 동적으로 크기를 조절할 수 있어 O(n)으로 삽입과 삭제가 가능
+ 접근 속도가 배열보다 느릴 수 있음
    - LinkedList의 경우 O(n)
+ 리스트의 종류
    - 내부 구현 방식에 따라 크게 두가지로 나뉜다
        + ArrayList (배열 기반)
            - 내부적으로 배열을 사용하여 구현된 리스트
            - 요소를 추가하면 내부 배열 크기를 자동으로 조정
                + 빈 공간이 있으면 O(1), 없으면 O(n)
            - 인덱스로 빠르게 접근 가능 (O(1))
            - 중간 삽입/삭제 시 요소 이동 필요 (O(n))
        + LinkedList
            - 각 요소(노드)가 포인터를 이용해 다음 요소와 연결
            - 삽입/삭제가 빠름 (O(1))
                + 중간에 추가해도 기존 요소를 이동할 필요가 없다
            - 인덱스 접근이 느림 (O(n))
                + 원하는 요소를 찾으려면 처음부터 탐색해야 함
            - 연결 리스트의 종류
                + 단일 연결 리스트
                + 이중 연결 리스트
                + 원형 연결 리스트

### 배열과 리스트의 차이
+ 배열과 리스트 모두 삽입과 삭제에 O(n)의 시간복잡도를 가진다
    - 그럼 둘 중 아무거나 골라서 사용해도 되나?
        + 배열의 삽입 O(n)
            - 요소를 한 칸씩 shift
        + 리스트의 삽입 O(n)
            - 원하는 위치까지 탐색
    - 만약에 삽입할 위치를 알고 있다면?
        + 배열은 위치를 알아도 요소를 이동시켜야 한다
        + 리스트는 위치를 알고 있으므로 포인터만 연결하면 된다

    |연산|배열|리스트|
    |---|---|---|
    |접근|O(1)|O(n)|
    |삽입/삭제 (위치 모를때)|O(n)|O(n)|
    |삽입/삭제 (위치 알 때)|O(n)|O(1)|

    - 정리
        + 배열이 유리한 경우
            - 데이터의 크기가 고정되어 있을 때
            - 빠른 조회가 필요할 때
        + 리스트가 유리한 경우
            - 삽입과 삭제가 빈번할 때
            - 삽입할 위치를 이미 알고 있을 때
