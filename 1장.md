## 자료구조 개요
+ 자료구조란 데이터를 효율적으로 저장하고 관리하는 방법을 의미
+ 효율적인 자료구조를 사용하면 데이터를 빠르게 검색, 삽입, 삭제할 수 있으며 알고리즘의 성능을 크게 향상시킬 수 있음

### 선형 자료구조
+ 데이터가 `순차적`으로 저장되는 구조로 인접한 데이터끼리 연결됨
    - 배열 (Array)
        + 같은 자료형의 데이터를 연속된 메모리에 저장
    - 리스트 (List)
        + 순차적으로 데이터를 저장하지만 동적으로 크기를 조절할 수 있음
        + 어레이리스트, 링크드리스트
    - 스택 (Stack)
        + 후입선출
    - 큐 (Queue)
        + 선입선출
    - 데크 (Deque)
        + 양쪽에서 삽입과 삭제가 가능한 자료구조
        + 슬라이딩 윈도우, 회문 검사 (Palindrome)
            - 회문 검사란 `앞에서 읽으나 뒤에서 읽으나 똑같은 문자열`
                + 기러기, 토마토, racecar, level 등
                    ```
                    "r a c e c a r"
                     ↑           ↑   → r == r ✅  계속
                       ↑       ↑     → a == a ✅  계속
                         ↑   ↑       → c == c ✅  계속
                           ↑         → 가운데 1개 남으면 회문 ✅
                    ```
                + 앞/뒤 양쪽에서 꺼내는게 O(1)이라 회문 검사에 딱 맞는 자료구조

### 비선형 자료구조
+ 데이터가 `계층적이거나 복잡한 관계`를 가지며 `비순차적`으로 저장됨
    - 해시 테이블 (Hash Table)
        + 키-값 형태로 데이터를 저장하고 해시 함수를 이용해 빠르게 검색이 가능
    - 그래프 (Graph)
        + 정점(Vertex)와 간선(Edge)로 구성되며 복잡한 관계를 표현하는데 사용
    - 트리 (Tree)
        + 부모-자식 관계를 가지는 계층적 자료구조
        + 이진트리, 이진탐색트리
    - 힙 (Heap)
        + 특정한 순서를 유지하는 트리 기반의 자료구조
        + 최소힙, 최대힙, 우선순위 큐

### 시간 복잡도 & 공간 복잡도
+ 시간 복잡도
    - 알고리즘이 실행되는데 걸리는 시간을 입력 크기(n)에 따라 분석한 것
    - 입력이 커질수록 얼마나 더 느려지는지 예측할 수 있게 도와줌
    - 빅-오 표기법 (Big-O)
        + 최악의 경우를 기준으로 연산 횟수를 표현
        + 안전한 성능 보장이 필요하기 때문에 최악을 기준으로 연산 횟수를 표현
            - O(1)
                + 상수 시간
                + 배열에서 인덱스로 접근
            - O(log n)
                + 로그 시간
                + 이진 탐색
                + 로그 log<sub>2</sub>(n)
            - O(n)
                + 선형 시간
                + 배열에서 특정 원소 찾기
            - O(n log n)
                + 로그 선형 시간
                + 퀵 정렬, 병합 정렬
                + O(n) 반복 * log<sub>2</sub>
            - O(n<sup>2</sup>)
                + 이차 시간
                + 버블 정렬, 선택 정렬
                + n(n+1)/2 (가우스 공식)
            - O(2<sup>n</sup>)
                + 지수 시간
                + 피보나치 수열의 재귀적 계산

            |입력 크기(n)|O(1)|O(log n)|O(n)|O(n log n)|O(n<sup>2</sup>)|
            |---|---|---|---|---|---|
            |10|1|3.3|10|33.2|100|
            |100|1|6.6|100|664.4|10000|
            |1000|1|9.9|1000|9966|1000000|
    - 상수가 버려지는 이유
        + 시간복잡도는 입력 n에 따라 성능이 어떻게 바뀌는지 보는 것
        + 이때 작은 수나 찾은 차수 항은 거의 영향이 없음
            ```
            T(n) = 3n<sup>2</sup> + 5n + 100
            - n이 작을 땐 모두 영향받지만 n이 커질수록 3n<sup>2</sup>의 영향력이 커짐
            - 그래서 O(n<sup>2</sup>)로 표기
            ```

+ 공간 복잡도
    - 알고리즘이 실행될 때 필요한 메모리 사용량을 분석한 것
    - 입력 데이터 크기에 따라 얼마나 많은 추가 공간이 필요한지 계산
    - 재귀 호출, 추가적인 배열 사용 등에 따라 공간 복잡도가 증가할 수 있음
    - 시간 복잡도만큼 공간도 제한적인 자원
        + CPU는 느려도 기다릴 수 있지만 메모리는 부족하면 `그대로 종료`
    - 시공간 트레이드 오프
        + 속도를 아끼기 위해 공간을 쓰기도 하고 공간을 아끼기 위해 속도를 포기하기도 함


    |구분|시간 복잡도|공간 복잡도|
    |---|---|---|
    |의미|연산 횟수|메모리 사용량|
    |줄이기 전략|연산 최적화, 반복문 개선|중복 데이터 제거|
    |충돌 가능성|느려짐|메모리 초과 에러|
